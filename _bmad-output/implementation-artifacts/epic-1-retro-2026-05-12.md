## What Went Well

1. **Layered architecture consistency** — Hexagonal layers (API → Application → Domain → Infrastructure) established in story 1-1 and maintained consistently across all 10 stories. Clear separation of concerns.

2. **Test-first backend** — 22 tests covering critical auth flows (success, failure, rate limiting, JWT claims, token rotation, logout). Tests catch real bugs.

3. **Code review with immediate fixes** — Story 1-4 code review found 8 issues (H1-H5, M1-M3) including type safety, hex validation, and import hygiene. All fixed before proceeding.

4. **Reusable patterns established** — ToastService with signals, AuthService, conftest.py engine lifecycle, audit logging with HMAC signatures — all ready for reuse in Epic 2+.

5. **Asset abstraction layer** — IAssetModule Protocol with runtime_checkable, event serialization/deserialization, idempotent handlers — clean foundation for vertical modules.

6. **Security-first auth** — Argon2id, RS256, HttpOnly cookies, rate limiting, anti-enumeration on password reset, audit logging with category=security.

## Challenges & Lessons Learned

### 1. Docker Test Environment Friction
**Problem:** The venv built with `uv` in the Dockerfile didn't include pip or dev dependencies. Required multiple container rebuilds to get tests running.  
**Root Cause:** Original Dockerfile installed only production deps (`uv pip install .`), not dev deps.  
**Fix Applied:** Changed to `uv pip install ".[dev]"` in Dockerfile.  
**Lesson:** Always include dev dependencies in the development Docker image from the start.

### 2. Audit Log Append-Only Trigger vs Test Cleanup
**Problem:** The `trg_audit_log_immutable` trigger on `audit_log` prevented DELETE, making test cleanup impossible.  
**Fix Applied:** Tests disable/enable trigger around cleanup (`ALTER TABLE audit_log DISABLE TRIGGER`).  
**Lesson:** Test infrastructure should account for production safety constraints. Consider a separate test database or transaction rollback strategy.

### 3. Redis State Leaking Between Test Runs
**Problem:** Rate limit keys persisted in Redis across test sessions, causing `test_login_nonexistent_user` to receive 429 instead of 401.  
**Fix Applied:** Changed `_clear_rate_limit(email)` to `_clear_all_rate_limits()` using `SCAN` + wildcard delete.  
**Lesson:** Always clean ALL external state (Redis, caches) in test fixtures, not just known keys.

### 4. Type Safety Afterthought
**Problem:** `LoginResult.user` was typed as `object`, causing a cascade of `type: ignore` comments in auth_routes.py.  
**Fix Applied:** Typed as `User | None` with `from __future__ import annotations`.  
**Lesson:** Type properly from the start. `object` and `Any` are code smells that propagate.

### 5. Frontend E2E Not Configured
**Problem:** Playwright was not installed, so E2E tests for login (AC10 of story 1-5) were deferred.  
**Impact:** No automated E2E coverage for auth flows.  
**Action Required:** Install and configure Playwright before Epic 2A front-end stories.
